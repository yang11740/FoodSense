# FoodSense 前端视觉设计方案

## 1. 设计定位

FoodSense 的整体视觉定位是：清新、亲切、圆润、活泼、可信。

产品不应呈现为严肃的医疗系统，也不应过度偏向健身数据工具。它更像一个移动端饮食健康小助手，帮助用户在吃饭前快速理解“这道菜是否适合我”“风险来自哪里”“我可以怎么调整”。

核心设计关键词：

- 清新绿色
- 圆润卡片
- 卡通亲切
- 低压力提醒
- 可解释建议
- 移动端轻操作

设计目标：

- 降低用户面对健康信息时的紧张感。
- 让拍照识别成为最突出的核心入口。
- 用颜色、标签、图标和短句解释复杂营养信息。
- 保持健康辅助产品的可信边界，不制造医疗诊断感。

## 2. 品牌主色与颜色系统

主色参考清新卡通绿色，接近小青菜、嫩叶、健康饮食的感受。

### 2.1 品牌色

| Token | 色值 | 用途 |
|---|---:|---|
| `--fs-green-50` | `#F0FBEF` | 页面浅绿背景、轻提示底色 |
| `--fs-green-100` | `#DCF8D8` | 绿色卡片底色、推荐标签底色 |
| `--fs-green-200` | `#CFF7A3` | 嫩芽辅助色、吉祥物辅助色 |
| `--fs-green-400` | `#6DDA7A` | 轻量按钮、图标背景 |
| `--fs-green-500` | `#4CCB63` | 品牌主绿、主按钮、拍照按钮 |
| `--fs-green-600` | `#16A34A` | 主按钮 hover、选中态文字 |
| `--fs-green-700` | `#15803D` | 强调文字、深色图标 |

### 2.2 辅助色

| Token | 色值 | 用途 |
|---|---:|---|
| `--fs-blue-500` | `#5BA7F7` | 报告、数据、同步、系统说明 |
| `--fs-yellow-400` | `#FFE88A` | 成就、轻量高亮、提示贴纸 |
| `--fs-orange-500` | `#FFB84D` | 谨慎食用、高盐/高油提醒 |
| `--fs-red-500` | `#FF6B6B` | 不推荐、明显风险、错误提示 |
| `--fs-purple-500` | `#A78BFA` | 睡眠、设备、画像扩展信息 |

### 2.3 中性色

| Token | 色值 | 用途 |
|---|---:|---|
| `--fs-bg` | `#F7FFF4` | App 全局背景 |
| `--fs-surface` | `#FFFDF7` | 卡片、底部导航、弹窗 |
| `--fs-surface-white` | `#FFFFFF` | 高密度内容卡片 |
| `--fs-text` | `#111827` | 主要文字 |
| `--fs-text-soft` | `#4B5563` | 正文说明 |
| `--fs-text-muted` | `#6B7280` | 辅助信息、单位、时间 |
| `--fs-border` | `#E5E7EB` | 细边框、分割线 |

### 2.4 状态色规则

| 状态 | 背景 | 文字 | 边框 | 使用场景 |
|---|---:|---:|---:|---|
| 推荐食用 | `#F0FBEF` | `#15803D` | `#BDEFC3` | 推荐结果、健康目标 |
| 谨慎食用 | `#FFF7E6` | `#B7791F` | `#FFD88A` | 高盐、高油、高糖等轻中风险 |
| 不推荐 | `#FFF1F1` | `#D94A4A` | `#FFC6C6` | 明确不推荐、严重风险 |
| 信息说明 | `#EFF7FF` | `#2563EB` | `#BFDBFE` | 系统说明、隐私提示、报告解释 |

红色只用于明确高风险，避免大面积使用。大多数饮食提醒优先使用橙色，保持友好和低压力。

## 3. 字体与排版

### 3.1 字体栈

```css
font-family: system-ui, -apple-system, BlinkMacSystemFont, "PingFang SC",
  "Microsoft YaHei", "Noto Sans CJK SC", sans-serif;
```

### 3.2 字号层级

| 层级 | 字号 | 字重 | 用途 |
|---|---:|---:|---|
| 页面标题 | `20px` | `700` | 首页、报告页、画像页标题 |
| 卡片标题 | `17px` | `700` | 识别结果、营养结构分析 |
| 小标题 | `15px` | `600` | 分组标题、表单标题 |
| 正文 | `15px` | `400` | 普通说明、建议内容 |
| 辅助文字 | `13px` | `400` | 时间、单位、解释语 |
| 标签文字 | `12px` | `600` | 风险标签、状态 Badge |
| 关键数字 | `24px-32px` | `700` | 健康评分、分析次数、趋势变化 |

### 3.3 文案语气

文案要像生活助手，不像医生诊断。

推荐表达：

- “这道菜今天有点重口噢”
- “可以吃，但建议少量”
- “搭配一份绿叶菜会更稳”
- “本周控盐表现不错”
- “高脂风险主要来自五花肉和红烧酱汁”

避免表达：

- “患者应避免摄入”
- “诊断为高风险”
- “严重不适合”
- “必须停止食用”

## 4. 圆角、阴影与布局

### 4.1 圆角系统

界面整体减少硬矩形，多使用圆角、胶囊和圆形。

| 元素 | 圆角 |
|---|---:|
| 大卡片 | `24px` |
| 小卡片 | `20px` |
| 图片容器 | `28px` |
| 主按钮 | `999px` |
| 标签 Badge | `999px` |
| 输入框 | `18px` |
| 底部导航 | 顶部 `28px` 圆角 |
| 弹窗 / 底部抽屉 | `28px` |

### 4.2 阴影

使用柔和绿色阴影，避免冷硬灰色阴影。

```css
--fs-shadow-card: 0 8px 24px rgba(76, 203, 99, 0.12);
--fs-shadow-soft: 0 4px 14px rgba(15, 23, 42, 0.06);
--fs-shadow-button: 0 10px 24px rgba(76, 203, 99, 0.28);
```

普通卡片：

```css
background: #FFFDF7;
border: 1px solid rgba(76, 203, 99, 0.12);
border-radius: 24px;
box-shadow: 0 8px 24px rgba(76, 203, 99, 0.12);
```

### 4.3 页面边距

移动端优先：

- 页面水平边距：`20px`
- 卡片内边距：`18px-20px`
- 卡片间距：`14px-16px`
- 底部导航高度：`72px-80px`
- 主要触控区域高度：不小于 `44px`

## 5. 图标与插画

### 5.1 图标风格

统一使用 `lucide-react` 线性图标。

建议参数：

```tsx
strokeWidth={1.75}
```

图标不要单独裸露太多，优先放在圆形或圆角浅色背景中。

### 5.2 核心图标映射

| 功能 | 图标 |
|---|---|
| 拍照识别 | `Camera` |
| 上传图片 | `ImagePlus` |
| 推荐食用 | `CheckCircle2` |
| 谨慎食用 | `AlertTriangle` |
| 不推荐 | `CircleX` 或 `OctagonAlert` |
| 营养结构 | `PieChart` |
| 数据趋势 | `BarChart3` / `TrendingUp` |
| 健康画像 | `UserRound` |
| 报告 | `FileText` |
| 提醒 | `Bell` |
| 隐私授权 | `ShieldCheck` |
| 手动修正 | `Pencil` |
| 历史记录 | `Clock3` |
| 饮食计划 | `CalendarDays` |

### 5.3 吉祥物方向

可以设计一个“小青菜饮食助手”作为品牌角色。

使用场景：

- 欢迎页
- 加载分析页
- 空状态
- 渐进式画像补全卡片
- 保存成功提示
- 周报鼓励语

角色气质：

- 绿色蔬菜形象
- 圆脸或圆润叶片
- 简单表情
- 不要医疗化，不穿白大褂

## 6. 核心组件规范

### 6.1 主按钮

用于拍照、查看详细分析、保存记录等主操作。

```css
background: #4CCB63;
color: white;
border-radius: 999px;
box-shadow: 0 10px 24px rgba(76, 203, 99, 0.28);
```

交互：

- hover / press 时颜色加深到 `#16A34A`
- active 时 `scale(0.96)`
- disabled 时透明度 `0.55`

### 6.2 次级按钮

用于“稍后再说”“修改识别结果”“导出数据”等。

```css
background: #F0FBEF;
color: #15803D;
border: 1px solid #BDEFC3;
border-radius: 999px;
```

### 6.3 状态标签

标签统一胶囊形，不使用尖锐矩形。

```css
border-radius: 999px;
padding: 6px 10px;
font-size: 12px;
font-weight: 600;
```

示例：

- 推荐食用：绿色
- 谨慎食用：橙色
- 不推荐：柔红色
- 高盐 / 高糖 / 高脂：橙色或红色
- 蛋白质 / 碳水 / 脂肪：使用营养色

### 6.4 卡片

卡片是主要信息容器，风格要像柔软的贴纸或便签。

卡片内容顺序：

1. 图标或状态
2. 标题
3. 关键结论
4. 简短解释
5. 可执行操作

避免在一个卡片中塞入过多数据。

### 6.5 底部导航

底部导航保持 5 个一级入口：

- 识别
- 分析
- 工具
- 报告
- 我的

视觉规则：

- 背景使用 `#FFFDF7`
- 顶部圆角 `28px`
- 当前项图标使用主绿
- 当前项可以轻微上浮 `translateY(-2px)`
- 当前项下方显示小圆点或短胶囊指示

## 7. 页面设计规范

### 7.1 首页 / 识别页

首页是产品第一视觉，应突出“拍照识别”。

布局顺序：

1. 顶部品牌区：FoodSense + 小青菜图形或圆形 Logo
2. 亲切主文案：“拍一下，看看适不适合你”
3. 大圆形拍照按钮
4. 辅助操作：上传图片、菜单截图
5. 今日小状态：已分析、本周风险、健康评分
6. 系统定位说明卡

拍照按钮建议：

- 尺寸：`96px-112px`
- 圆形
- 主绿填充
- 白色相机图标
- 外圈柔和绿色光晕
- 点击时轻微弹跳

### 7.2 菜品分析页

分析页遵循“先结论，后解释”。

布局顺序：

1. 菜品图片
2. 菜名 + 烹饪方式
3. 结论卡：推荐 / 谨慎 / 不推荐
4. 风险来源解释
5. 营养结构图表
6. 主要食材标签
7. 个性化建议
8. 免责声明

结论卡文案示例：

- 推荐：“这道菜比较适合今天的目标”
- 谨慎：“可以吃，但建议控制份量”
- 不推荐：“今天不太建议多吃噢”

风险解释要说明来源：

- “高脂风险主要来自五花肉”
- “钠摄入风险可能来自酱汁和腌制调味”
- “糖分风险可能来自糖醋汁”

### 7.3 健康画像页

画像页不做医疗档案感，而做“我的健康小地图”。

布局顺序：

1. 用户昵称与画像等级
2. 画像成长进度：L0 / L1 / L2 / L3
3. 3D 人体或健康状态图
4. 健康目标标签
5. 当前关注项
6. 轻量补全入口
7. 隐私与授权说明

画像进度文案：

- “你的饮食画像正在成长”
- “再补充 2 项，就能给出更贴合的建议”
- “可以随时跳过或修改”

### 7.4 报告页

报告页做成“本周饮食小结”，避免后台仪表盘感。

布局顺序：

1. 本周一句话总结
2. 关键指标卡：风险次数、健康评分、连续记录
3. 风险趋势图
4. 高频风险来源
5. 下周小目标
6. 提醒设置入口

总结文案示例：

- “这周高盐次数少了 2 次，做得不错”
- “油炸类有点多，下周可以控制在 2 次以内”
- “蔬菜搭配更稳定了”

### 7.5 工具页

工具页保持清晰列表，不做复杂宫格。

推荐顺序：

1. 营养分析记录
2. 健康报告
3. 饮食风险记录
4. 用药与饮食提醒
5. 设备健康数据
6. 数据统计分析
7. 饮食计划
8. 设置与隐私

每个工具项：

- 左侧圆角图标底
- 中间标题与一句说明
- 右侧箭头
- 卡片圆角 `20px`

### 7.6 渐进式画像补全卡片

这是产品核心体验之一。

触发场景：

- 多次识别到高盐菜品
- 多次识别到高糖菜品
- 用户查看报告
- 用户保存记录后

卡片样式：

- 使用蓝色或浅绿信息卡
- 不使用红色警告样式
- 提供明确跳过按钮

文案示例：

“最近 7 天识别到 4 次高盐菜品。你是否关注控盐或血压？”

按钮：

- “关注控盐”
- “暂不回答”

## 8. 图表与数据可视化

### 8.1 营养结构颜色

| 营养项 | 色值 |
|---|---:|
| 碳水 | `#5BA7F7` |
| 蛋白质 | `#4CCB63` |
| 脂肪 | `#FFB84D` |
| 钠 | `#A78BFA` |
| 糖 | `#FF8FAB` |
| 热量 | `#FF6B6B` |

### 8.2 图表风格

- 图表线条使用圆角。
- 网格线使用浅灰 `#E5E7EB`，透明度降低。
- 图表 Tooltip 使用圆角 `16px`。
- 不使用过深、过硬的坐标轴。
- 移动端优先显示结论，再显示图表。

## 9. 交互动效

动效要轻快、软弹、短时长。

| 场景 | 动效 |
|---|---|
| 拍照按钮点击 | `scale(0.96)` 后回弹 |
| 分析中 | 圆环扫描 + 阶段文案变化 |
| 结果出现 | 卡片从下方淡入并轻微上移 |
| 标签出现 | 依次轻弹入场 |
| 营养进度条 | 从 0 增长到目标值 |
| 健康分变化 | 数字滚动 |
| 底部导航切换 | 图标上浮 + 颜色变化 |
| 保存成功 | 绿色波纹 + 勾选图标 |
| 吉祥物状态 | 眨眼、轻微点头、轻微漂浮 |

推荐时长：

- 微交互：`120ms-180ms`
- 卡片入场：`220ms-300ms`
- 图表 / 进度条：`500ms-700ms`

缓动：

```css
transition-timing-function: cubic-bezier(0.2, 0.8, 0.2, 1);
```

## 10. 免责声明与可信边界

FoodSense 必须保持健康辅助边界，避免医疗化承诺。

免责声明卡片视觉：

- 使用浅蓝或浅绿
- 圆角 `20px`
- 图标使用 `Info` 或 `ShieldCheck`
- 文案短而清楚

推荐文案：

“分析结果用于饮食风险感知与辅助决策，不构成医疗诊断或治疗建议。最终饮食选择请结合个人实际情况判断。”

不要使用强制性、诊断性表述。

## 12. 实施优先级

第一阶段：

- 替换主题色为清新绿色。
- 全局背景改为浅绿。
- 卡片圆角改为 `20px-24px`。
- 按钮改为胶囊形。
- 首页拍照按钮改为大圆形绿色按钮。

第二阶段：

- 调整首页、分析页、报告页的信息层级。
- 状态标签改成软糖胶囊风格。
- 报告页改成“本周饮食小结”表达。
- 渐进式画像补全卡片上线。

第三阶段：

- 加入小青菜吉祥物或插画。
- 加入分析中、保存成功、空状态动效。
- 优化 3D 健康画像页的亲切表达。
