# FoodSense 系统架构

## 一、系统架构总览

FoodSense 是一个面向健康饮食管理的智能饮食决策辅助系统，整体采用“前端应用 + 后端服务 + 本地数据库 + AI 能力服务”的架构。系统围绕用户账户、AI 食物识别、智能对话助手、饮食记录、健康分析报告与 3D 健康画像等核心功能展开。

### 1. 架构分层

| 层级 | 组成 | 主要职责 |
| --- | --- | --- |
| 用户交互层 | Web 前端、桌面端、移动端页面 | 展示登录注册、首页、拍照识别、聊天助手、历史记录、健康报告、3D 画像等界面 |
| 前端应用层 | React 页面、组件、状态与工具函数 | 处理用户操作、图片上传、数据展示、图表渲染、3D 模型展示、调用后端 API |
| 后端服务层 | Express API 服务 | 提供用户认证、资料管理、饮食记录、健康分析、聊天记录、AI 识别等接口 |
| 数据存储层 | SQLite 数据库 | 存储用户账号、个人资料、饮食记录、健康目标、饮食偏好、聊天消息等数据 |
| AI 能力层 | 大模型文本能力、视觉识别能力 | 完成食物图片识别、营养估算、饮食建议生成、智能助手回复等能力 |
| 分析决策层 | 健康分析、营养目标、身体部位影响评估 | 根据用户资料和饮食记录生成健康摘要、周报、风险提示和 3D 画像状态 |

### 2. 核心业务流程

#### 用户认证与资料初始化

用户通过注册或登录进入系统，填写年龄、性别、身高、体重、饮食方式、健康目标等基础信息。系统将这些信息作为后续饮食推荐、健康分析和智能助手回复的个性化依据。

#### AI 视觉识别与饮食决策

用户上传或拍摄食物图片后，前端将图片数据提交给后端。后端结合用户资料、健康目标、历史饮食记录调用视觉识别模型，返回食物名称、热量、碳水、蛋白质、脂肪、食材、烹饪方式、风险标签和饮食建议。

#### 智能对话助手「小膳青」

用户可以通过聊天页面向“小膳青”咨询饮食建议、食物选择、健康习惯和营养问题。系统会结合用户资料、饮食记录、目标和偏好生成个性化回复，并保存对话历史。

#### 饮食记录与历史回顾

用户可以保存 AI 识别结果，也可以管理历史饮食记录。系统按日期、餐次、热量和营养指标组织数据，支持后续健康统计、趋势分析和报告生成。

#### 健康分析与 3D 画像

系统根据近期饮食记录生成健康摘要、风险趋势、饮食平衡评分、下周目标和提醒建议。3D 画像模块通过人体模型展示健康状态，并结合头部、心脏、肠胃、骨骼、肌肉、皮肤等部位给出饮食影响分析。

### 3. 数据流向

```text
用户操作
  ↓
React 前端页面
  ↓
Express 后端 API
  ↓
SQLite 本地数据库
  ↓
健康分析模块 / AI 识别模块 / 智能对话模块
  ↓
返回识别结果、饮食建议、历史记录、健康报告、3D 画像状态
  ↓
前端可视化展示
```

## 二、技术栈

### 1. 前端技术栈

| 技术 | 用途 |
| --- | --- |
| React | 构建前端页面和组件 |
| TypeScript | 提升前端代码类型安全和可维护性 |
| Vite | 前端开发服务器与项目构建工具 |
| Tailwind CSS | 页面样式与响应式布局 |
| Radix UI | 下拉菜单、弹窗、标签页、开关、选择器等基础交互组件 |
| Material UI Icons | 页面图标资源 |
| lucide-react | 轻量图标库 |
| Recharts | 健康数据、营养趋势和统计图表展示 |
| Three.js | 3D 人体画像渲染能力 |
| @react-three/fiber | 在 React 中使用 Three.js |
| @react-three/drei | 3D 模型加载、相机控制等辅助能力 |
| GLB 3D 模型 | 展示人体健康画像 |
| Capacitor | 支持构建 Android 移动端应用 |
| Electron | 支持构建桌面端应用 |

### 2. 后端技术栈

| 技术 | 用途 |
| --- | --- |
| Node.js | 后端运行环境 |
| Express | 构建 REST API 服务 |
| SQLite | 本地关系型数据库 |
| sqlite3 | Node.js 访问 SQLite 数据库 |
| bcryptjs | 用户密码加密与登录校验 |
| cors | 处理前后端跨域请求 |
| dotenv | 管理后端环境变量 |
| nodemon | 后端开发时自动重启服务 |

### 3. AI 与智能分析能力

| 能力 | 说明 |
| --- | --- |
| 文本大模型 | 用于生成“小膳青”聊天回复、饮食建议和健康解释 |
| 视觉大模型 | 用于识别食物图片、营养标签、外卖订单等图像内容 |
| OpenAI 兼容接口 | 后端通过统一接口调用大模型服务 |
| 默认模型配置 | 文本模型默认 `glm-4-flash`，视觉模型默认 `glm-4v-flash` |
| 健康分析模块 | 根据饮食记录生成健康摘要、风险趋势、周报和身体部位影响 |
| 营养目标模块 | 根据用户资料生成每日热量和营养摄入参考 |

### 4. 部署与运行方式

| 运行形态 | 说明 |
| --- | --- |
| Web 开发环境 | 前端默认运行在 `http://localhost:5173` |
| 后端 API 服务 | 后端默认运行在 `http://localhost:4001` |
| 本地数据库 | SQLite 数据文件位于 `backend/foodsense.db` |
| 桌面端 | 通过 Electron 打包为桌面应用 |
| 移动端 | 通过 Capacitor 同步并构建 Android 应用 |

## 三、数据库表设计

当前系统使用 SQLite 数据库，启动后端服务时会自动创建所需数据表。数据库文件为：

```text
backend/foodsense.db
```

### 1. users 用户表

用于存储用户账号信息。

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 用户 ID |
| name | TEXT | NOT NULL | 用户昵称 |
| email | TEXT | NOT NULL, UNIQUE | 用户邮箱 |
| password | TEXT | NOT NULL | 加密后的用户密码 |
| created_at | TEXT | DEFAULT CURRENT_TIMESTAMP | 注册时间 |

### 2. profiles 用户健康资料表

用于存储用户基础健康资料、饮食状态和提醒设置。

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 资料 ID |
| user_id | INTEGER | NOT NULL, UNIQUE, FOREIGN KEY | 所属用户 ID |
| goal | TEXT |  | 用户主要健康目标 |
| age | TEXT |  | 年龄 |
| gender | TEXT |  | 性别 |
| dietStyle | TEXT |  | 饮食方式或饮食风格 |
| mood | TEXT |  | 当前状态或情绪 |
| height | TEXT |  | 身高 |
| weight | TEXT |  | 体重 |
| reminder_settings | TEXT |  | 提醒设置，JSON 字符串 |
| updated_at | TEXT | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

### 3. recipe_records 饮食记录表

用于保存用户每次饮食识别或手动记录的结果。

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | TEXT | PRIMARY KEY | 饮食记录 ID |
| user_id | INTEGER | NOT NULL, FOREIGN KEY | 所属用户 ID |
| date | TEXT | NOT NULL | 饮食日期 |
| meal | TEXT | NOT NULL | 餐次，例如早餐、午餐、晚餐、加餐 |
| name | TEXT | NOT NULL | 食物或菜品名称 |
| calories | INTEGER | NOT NULL, DEFAULT 0 | 热量 |
| carbs | INTEGER | NOT NULL, DEFAULT 0 | 碳水化合物 |
| protein | INTEGER | NOT NULL, DEFAULT 0 | 蛋白质 |
| fat | INTEGER | NOT NULL, DEFAULT 0 | 脂肪 |
| ingredients | TEXT | NOT NULL, DEFAULT '[]' | 食材列表，JSON 字符串 |
| cookingTechnique | TEXT | NOT NULL | 烹饪技法 |
| cookingMethod | TEXT | NOT NULL | 烹饪方式说明 |
| recommendation | TEXT | NOT NULL | 饮食建议结果 |
| summary | TEXT | NOT NULL | 本次饮食总结 |
| reasons | TEXT | NOT NULL, DEFAULT '[]' | 建议原因，JSON 字符串 |
| tags | TEXT | NOT NULL, DEFAULT '[]' | 风险或营养标签，JSON 字符串 |
| created_at | TEXT | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TEXT | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

索引：

| 索引名 | 字段 | 说明 |
| --- | --- | --- |
| idx_recipe_records_user_date | user_id, date | 提升按用户和日期查询饮食记录的速度 |

### 4. user_goals 用户健康目标表

用于存储用户设置的多个健康目标。

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 目标 ID |
| user_id | INTEGER | NOT NULL, FOREIGN KEY | 所属用户 ID |
| goal | TEXT | NOT NULL | 健康目标内容 |
| created_at | TEXT | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

唯一约束：

| 字段 | 说明 |
| --- | --- |
| user_id, goal | 同一用户不能重复添加相同目标 |

### 5. user_preferences 用户饮食偏好表

用于存储用户饮食偏好、忌口或偏爱的饮食方式。

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 偏好 ID |
| user_id | INTEGER | NOT NULL, FOREIGN KEY | 所属用户 ID |
| preference | TEXT | NOT NULL | 饮食偏好内容 |
| created_at | TEXT | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

唯一约束：

| 字段 | 说明 |
| --- | --- |
| user_id, preference | 同一用户不能重复添加相同饮食偏好 |

### 6. chat_messages 智能助手聊天消息表

用于保存用户与「小膳青」的对话历史。

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | TEXT | PRIMARY KEY | 消息 ID |
| user_id | INTEGER | NOT NULL, FOREIGN KEY | 所属用户 ID |
| role | TEXT | NOT NULL | 消息角色，用户或助手 |
| text | TEXT | NOT NULL | 消息内容 |
| created_at | TEXT | DEFAULT CURRENT_TIMESTAMP | 发送时间 |

索引：

| 索引名 | 字段 | 说明 |
| --- | --- | --- |
| idx_chat_messages_user_created | user_id, created_at | 提升按用户和时间读取聊天记录的速度 |

## 四、主要接口模块

| 模块 | 接口示例 | 说明 |
| --- | --- | --- |
| 健康检查 | `GET /api/ping` | 检查后端服务是否正常 |
| 用户认证 | `POST /api/auth/register`、`POST /api/auth/login` | 注册和登录 |
| 用户资料 | `GET /api/profile`、`POST /api/profile`、`POST /api/user` | 获取和更新用户资料 |
| 饮食记录 | `GET /api/recipes`、`POST /api/recipes` | 查询和保存饮食记录 |
| 健康目标 | `GET /api/goals`、`POST /api/goals`、`DELETE /api/goals` | 管理用户健康目标 |
| 饮食偏好 | `GET /api/preferences`、`POST /api/preferences`、`DELETE /api/preferences` | 管理饮食偏好 |
| 健康分析 | `GET /api/health/summary`、`GET /api/health/profile` | 获取健康摘要和 3D 画像分析数据 |
| 营养目标 | `GET /api/nutrition/targets` | 获取每日营养摄入参考 |
| AI 视觉识别 | `POST /api/analyze` | 识别食物图片并生成饮食建议 |
| 智能助手 | `GET /api/chat/messages`、`POST /api/chat` | 获取聊天历史和发送消息 |
| 提醒设置 | `GET /api/reminders/settings`、`POST /api/reminders/settings` | 获取和保存提醒设置 |

## 五、系统特点

- 以用户个人资料、饮食目标和历史记录为基础，提供个性化饮食建议。
- 支持拍照识别食物，并给出热量、营养成分、风险标签和食用建议。
- 内置智能对话助手「小膳青」，可持续陪伴用户进行饮食咨询。
- 饮食记录可沉淀为长期健康数据，支持历史回顾和趋势分析。
- 健康报告结合图表、提醒、目标和 3D 画像，让用户更直观看到饮食对身体状态的影响。
- 支持 Web、桌面端和 Android 移动端多形态运行。
